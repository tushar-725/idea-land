# Transformation Tracker

<span class="doc-badge">Idea Overview</span>

## Idea Summary

Transformation Tracker is a product idea focused on helping users track visible progress over time through structured photo-based journeys.

Core flow:

> Open -> Capture -> Save.

The idea currently has three core documentation tracks:

- **Change Tracker**: full product specification and positioning.
- **AI Roadmap**: phased AI strategy and boundaries.
- **Phase 1 Features**: concrete launch scope for execution.

## Documentation Structure

### 1. Change Tracker

Defines:

- Product problem and positioning
- Core concepts (journeys, entries, goals, streaks)
- Feature set and MVP baseline
- Long-term vision

Open: [Change Tracker](./change-tracker.md)

### 2. AI Roadmap

Defines:

- AI principles and product constraints
- Phase 1, 2, and 3 capabilities
- Explicit exclusions (medical diagnosis, exact measurements)
- Feasibility and user-value framing

Open: [AI Roadmap](./ai-roadmap.md)

### 3. Phase 1 Features

Defines:

- Launch categories (hair, skin, physique)
- Non-AI baseline for phase execution
- Lightweight AI additions for phase 1
- Success criteria and out-of-scope areas

Open: [Phase 1 Features](./phase1-features.md)
